## By User Type

### Business Operators

You run a business and want to use AI to optimize operations.

**Start here:** [Business Guide](getting-started/business-guide.md)

**Key missions:**
- `/coord optimize` - Find your biggest opportunity
- `/coord marketing-system-setup` - Build marketing operations
- `/coord sales-system-setup` - Build sales operations

**Relevant docs:**
- [Business Chassis](concepts/business-chassis.md)
- [Operations Library](OPERATIONS-LIBRARY.md)
- [Quick Reference](reference/quick-reference.md)

### Product Developers

You're building software products with BOS-AI + AGENT-11.

**Start here:** [Developer Guide](getting-started/developer-guide.md)

**Key missions:**
- Foundation documents (vision, market research, positioning)
- `/coord prd-creation` - Create product requirements
- Hand off PRD to AGENT-11 for development

**Relevant docs:**
- [Lifecycle](concepts/lifecycle.md)
- [Foundation Documents](Document%20Library/Foundation/)
- [Repository Structure](contributing/repository-structure.md)

### New Users

You're just starting and not sure which path to take.

**Start here:** [First Mission](getting-started/first-mission.md)

Then run `/coord optimize` - the system will guide you based on your specific situation.

## Essential Commands

| Command | Purpose |
|---------|---------|
| `/coord` | Interactive mission menu |
| `/coord optimize` | Business Chassis analysis |
| `/coord [mission]` | Run specific mission |
| `/meeting @agent "topic"` | Direct agent consultation |
| `/report` | Progress report |
| `/pmd "issue"` | Root cause analysis |

[Full Command Reference](reference/quick-reference.md)

## Documentation Standards

All BOS-AI documentation follows these principles:

1. **Progressive Disclosure** - Start simple, link to details
2. **Two Personas** - Business operators and product developers
3. **Actionable** - Every doc includes specific commands and next steps
4. **Cross-linked** - Every doc links to related content
5. **Maintained** - Updated with each BOS-AI release
